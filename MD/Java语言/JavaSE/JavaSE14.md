4.异常
	* 概念
		我们在写代码的时候,经常出现的一些小问题,为了方便我们处理这些异常,Java为我们提供了异常机制
		一个异常包含了错误的类型,原因以及位置
		异常:不正常,我们在代码的时候出现的编译或者运行的错误
	* 体系
		Throwable:(是所有异常的父类,也只有继承自这个类才能被throw抛出)
			方法:
				String getMessage():原因
				String toString():类型和原因
				Void printStackTrace():类型原因和位置
			|------->Exception(可以处理的一些问题)
					|--->RuntimeException(运行时期出现的异常)
					|--->非RuntimeExpection(编译时异常):特点在编译时期必须处理
			|---->Error(不能处理的严重问题 )	
	* 异常的分类:
			运行时期异常:RuntimeException的子类就是运行时期异常
			编译时期异常:在编译时期必须处理
	* 异常的处理方式:
		捕获处理:
			Try{
				有可能出现问题的代码;//这行代码出现异常后,底层会创建一个异常对象new RuntimeException
			}catch(异常类型 变量名){ //这行代码会捕获()中类型的底层创建的异常对象,如果未匹配到对应类型的异常对象,则会把异常抛给Jvm
				处理方式:
			}finaly{
				异常出现后还需要执行的代码;
			}
			Try..catch的执行顺序:
					首先执行try语句,如果发现异常,异常下面的代码不在执行,直接跳到catch语句中,catch语句结束后,整个trycatch语句结束,
					如果没有发现异常,则try语句执行结束后则整个trycatch语句执行完成,不会执行catch异常
		抛异常:
			throw处理,当我们不想处理异常,或者没有能力处理的时候,我们可以选择抛出异常,谁调用方法谁就处理异常.
			原则:不能交给Jvm处理
		Jvm处理异常的方式:
					如果出现异常我们没有处理,Jvm会帮我们进行处理,它会吧异常信息打印出来,然后终止程序,异常后面的代码将不在执行.			
		多个异常的处理方式:
						可以使用多个trycatch语句进行捕获
						使用一个trycatch之间的关系
					异常处理的顺序:
						多个catch之间可以有子父类
						评级之间没有顺序
						如果有子父类的时候父类必须要放在后面
	* Throw :
			制造异常的方式,并且结束方法,位置在:方法中,后面跟的是一个对象
	* Throws:
			处理异常的一种方式,将异常抛出,位置在:方法声明上,后面跟的是异常类型,并且可以跟多个
	注意:如果抛出的是编译时期异常,必须在方法声明处抛出
	如何自定义异常:
				写一个类去继承Exception或者RuntimeException,然后实现多个构造
